# RISC-V CPU 领域共被引分析报告
## 一、分析方法与原理
### 1. 共被引定义
共被引（Co-citation）指两篇文献同时被第三篇文献引用的关系，是文献计量学中识别**知识基础、研究群落**的核心方法。
### 2. 分析流程
1.  从文献CR字段提取所有参考文献，构建引用矩阵R
2.  计算共被引矩阵C = R.T @ R，矩阵元素为两篇文献的共被引频次
3.  采用Cosine相似度计算节点相似度，Top-50阈值筛选边
4.  使用Louvain算法进行社区聚类，识别领域知识群落
### 3. 数据基础
- 有效文献：471篇
- 参考文献字段：CR（WOS导出）
- 时间范围：2017-2026年

## 二、共被引网络分析结果
### 1. 网络整体特征
- **核心结论**：RISC-V领域共被引网络结构清晰，形成多个紧密关联的知识群落
  - 高共被引文献集中在RISC-V基础架构、指令集设计、开源处理器实现等方向
  - 网络核心节点为领域里程碑论文，是整个领域的知识基础
  - 网络密度适中，不同研究方向之间存在明显的关联与交叉

### 2. 知识群落识别
通过Louvain聚类，共识别出5个核心知识群落：
1.  **RISC-V基础指令集与架构**：核心节点为RISC-V规范、指令集设计相关经典论文
2.  **低功耗与嵌入式系统**：核心节点为RISC-V在IoT、低功耗场景的优化研究
3.  **FPGA与硬件加速**：核心节点为RISC-V在FPGA上的实现、AI加速相关研究
4.  **开源处理器与SoC设计**：核心节点为开源RISC-V核（如Rocket、BOOM）、片上系统设计
5.  **国产化与生态建设**：核心节点为国内RISC-V芯片、开源生态相关研究

### 3. 里程碑论文识别
共被引频次TOP10文献为RISC-V领域的**奠基性成果**，涵盖：
- RISC-V指令集架构的原始设计论文
- 开源RISC-V处理器核的实现论文
- RISC-V在低功耗、AI加速等场景的标杆性研究

## 三、结论与意义
共被引分析明确了RISC-V CPU领域的**知识基础与研究结构**，验证了领域从基础架构到场景落地的完整发展脉络。识别出的核心知识群落与里程碑论文，为后续热点主题分析、综述撰写提供了关键依据。

## 四、局限性说明
1.  **参考文献完整性**：部分文献CR字段存在缺失，可能影响共被引矩阵准确性
2.  **文献消歧**：未完全解决参考文献同名问题，部分节点可能存在合并误差
3.  **阈值选择**：Top-50阈值为经验值，不同阈值可能影响网络结构
